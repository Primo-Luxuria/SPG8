Short Answer

11.What is the difference between malloc and calloc?

Answer: malloc(size) allocates a single block of uninitialized memory of size bytes; calloc(n, size) allocates an array of n elements, each of size bytes, and initializes all bits to zero.

Grading: 3 pts for stating that malloc allocates uninitialized memory and calloc zero-initializes 2 pts for noting that calloc takes two parameters for number of elements and element size

12.How does the sizeof operator behave when applied to an array parameter inside a function?

Answer: Inside a function, an array parameter “decays” to a pointer, so sizeof yields the size of the pointer type (e.g. 8 on a 64-bit system), not the total array length.

Grading: 3 pts for explaining that arrays decay to pointers in parameter lists 2 pts for stating that sizeof then returns the pointer’s size

13.Explain what happens when you call free() on a NULL pointer.

Answer: The C standard guarantees that free(NULL) does nothing; it is a safe no-op.

Grading: 3 pts for saying “no effect” or “no operation” 2 pts for citing that this behavior is defined by the C standard

Essay

14.Discuss the significance of pointers in C in terms of performance and flexibility, citing examples where pointer arithmetic is beneficial and potentially dangerous.

Answer: Performance: passing large structs by pointer avoids copying Flexibility: implementing data structures (linked lists, trees) Pointer arithmetic: Beneficial: iterating arrays efficiently Dangerous: off-by-one errors lead to undefined behavior Best practices: boundary checks, use of const

Grading: 5 pts for discussing performance gains 5 pts for illustrating flexibility in data structures 5 pts for balanced examples of benefits vs. dangers 5 pts for overall coherence and depth

15.Describe the lifetime and scope of variables in C, including automatic, static, and dynamic storage.

Answer: Key Points: Scope: block (local), file (global), function prototype Lifetime/Storage Duration: Automatic: exists only within block; allocated on stack Static: persists for entire program; either global or inside function with static keyword Dynamic: allocated/freed via malloc/free on the heap

Grading: 5 pts for clearly defining scope levels 5 pts for explaining each storage duration 5 pts for providing illustrative examples 5 pts for clarity, organization, and completeness
